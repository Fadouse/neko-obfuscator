1. APPROVED BARRIER-SAFE HELPERS (definition + usage)
================================================================================

The strict-no-JNI runtime exposes the following helpers for any oop / Klass /
ConstMethod load. Every emitter that needs to read a Java VM pointer routes
through one of these:

  neko_load_oop_from_cell        (compressed/uncompressed oop load from a JNI
                                  handle cell; tag-bit aligned)
  neko_decode_klass_pointer      (compressed Klass* decode using NekoVmLayout
                                  klass_offset_base + shift)
  neko_class_klass_pointer       (Class.klass field via VMStructs offset)
  neko_method_const_method       (Method::_constMethod via VMStructs offset)
  neko_pending_exception         (Thread::_pending_exception read)
  neko_set_pending_exception     (Thread::_pending_exception write — raw oop
                                  store, atomic; replaces env->Throw on the
                                  hot path per M5b Oracle-guided fix)
  neko_clear_pending_exception   (atomic store NULL)

3. EXEMPT SITES (with explicit justification)
================================================================================

The following sites perform low-level pointer manipulation but are gated and
therefore exempt from the FORBIDDEN classification:

  BootstrapEmitter.java:2335-2358 — Oracle 9 bootstrap-only allowlist
      Calls: FindClass, GetMethodID, NewStringUTF, NewObjectA, NewGlobalRef,
             DeleteLocalRef.
      Gate:  Invoked exclusively from JNI_OnLoad via the throwable-cache init
             path (bootstrap_throwable_cache).
      Reason: Oracle 9 Fix K (commit 3934df6 + 906c811) explicitly authorizes
             this allowlist for bootstrap-time exception oop allocation +
             global-ref publication. The cached jthrowable globals are then
             read by neko_raise_cached_pending() on the hot path WITHOUT any
             additional JNI call.

  BootstrapEmitter.java:2397 — single env->Throw site
      Call:  (*env)->Throw(env, cached);
      Gate:  Helper neko_throw_cached(env, cached) — not invoked from translated
             methods. Translated paths use neko_set_pending_exception with the
             raw oop decoded from the cached jthrowable global (M5b learning,
             commit 5e5dd83). neko_throw_cached is reserved for the bootstrap
             return path.
      Reason: Oracle 9 Fix K invariant. Single env->Throw use point repository-wide.

  BootstrapEmitter.java:2400-2404 — neko_raise_cached_pending oop decode
      Code: cell = ((uintptr_t)cached) & ~(uintptr_t)(sizeof(void*) - 1u);
            void *oop = neko_load_oop_from_cell(cell);
      Reason: Tagged JNI handle alignment + barrier-safe decode (M5b pattern).
             Decoded oop is then atomically stored to Thread::_pending_exception
             via neko_set_pending_exception — no env-> call on hot path.

  Wave2FieldLdcEmitter.java LDC handlers — string root publication
      Reason: Reads cached_global from the global-ref published at bootstrap;
             never allocates new globals on hot path. Reads gated by
             g_neko_string_root_backend == NEKO_STRING_ROOT_BACKEND_BOOTSTRAP
             (see JniOnLoadEmitter prewarm publication).
